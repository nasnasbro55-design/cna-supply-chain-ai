# Contributing Guide

## Branch Naming Convention
- Your feature work: `feature/your-name-description`
  - Example: `feature/nasrullah-map-component`
  - Example: `feature/karina-traffic-api`
- Bug fixes: `fix/description`

## Workflow
1. Pull latest from `main` before starting work
2. Create your feature branch
3. Commit regularly with clear messages
4. Push your branch and open a Pull Request to `main`
5. Ethan reviews and merges

## Commit Message Format
- `add map component with leaflet`
- `fix traffic API connection timeout`
- `train NLP classifier on CrisisMMD`

## NEVER commit
- API keys or secrets
- .env files
- node_modules/
- Large model weight files (.pt, .pkl, .h5)

## Questions / Blockers
Post in the team group chat immediately if stuck.
Don't sit on a blocker for more than 24 hours.
